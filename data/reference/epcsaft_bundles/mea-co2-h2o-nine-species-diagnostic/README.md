# Nine-species MEA diagnostic Provider bundle

Version `1/` is generated by `scripts/build_mea_diagnostic_bundle.py` from the retained Phase 2 parameter tables. It is an application-owned `user-provided` Provider bundle, not an installed Provider catalog entry.

The artifact is diagnostic and nonpredictive. In particular, the MEAH+ and MEACOO- records remain provisional historical evaluation inputs, and the transferred trace-ion records are not independent MEA-system fits. See `docs/ePC-SAFT/full-component-parameter-source-audit.md` for the parameter-by-parameter evidence limits.

The receipt's fixed-density state proves finite public EOS evaluation and native
SDK transport only. Its pressure is negative, so it is not a mechanically valid
equilibrium state or physical-acceptance sentinel.

Build it only with the exact Provider wheel recorded in `1.receipt.json`:

```bash
PYTHONPATH=src /path/to/exact-provider-python \
  scripts/build_mea_diagnostic_bundle.py \
  --output data/reference/epcsaft_bundles/mea-co2-h2o-nine-species-diagnostic/1 \
  --receipt data/reference/epcsaft_bundles/mea-co2-h2o-nine-species-diagnostic/1.receipt.json \
  --provider-wheel /path/to/exact/epcsaft.whl
```

The command refuses to overwrite either output. Reload the directory with `ParameterBundle.from_path(...)`, select the exact component order from `MEA.epcsaft_ionic.diagnostic_bundle.COMPONENT_IDS`, and compare both fingerprints with the receipt before constructing `EPCSAFT`.
